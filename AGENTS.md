# Planner Repository Instructions

## Session Start Requirements

Before substantive planning, implementation, or review work:

1. Read `docs/session-start-and-doc-index.md`.
2. Read `.codex/project-skill-config.md`.
3. Read the always-load docs listed there.
4. Read `docs/project-plan.md`.

## Workflow Model

- Planner now uses the generalized three-skill lifecycle.
- Start with `.codex/skills/project-bootstrap/`.
- Use `.codex/skills/spec-lifecycle/` for planning, spec, and plan-synchronization work.
- Use `.codex/skills/delivery-cycle/` for implementation, verification, review, and closeout.
- Do not start meaningful implementation unless the relevant plan or spec is ready enough to support bounded execution.

## Documentation Rules

- Keep `docs/session-start-and-doc-index.md` current when adding durable planning or architecture docs.
- Keep `docs/project-plan.md` aligned with the active planning thread.
- Prefer small, explicit planning updates over scattered undocumented decisions.
